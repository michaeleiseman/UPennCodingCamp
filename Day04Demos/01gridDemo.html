Learning Grid

An html grid element typically contains elements that need to be precisely positioned in both horizontal rows and vertical columns. Giving a value of grid to an element's CSS display property enables control of the 2-dimensional spacial arrangement of the items within it. Properties given to the style attribute of the grid control how the nested elements are positioned. In the following examples, we'll see what these properties are and what they do.

We will show how to arrange grids using two different methods:(Click on the triangles on the left to see the examples.)

▸The first method is useful for relatively simple grids where each column runs down the complete grid uninterupted, and each row runs across the entire grid uninterupted. Not all of the rows need to have the same height, nor do all of the columns need to have the same width.

The grid-template-columns property of the grid element defines the structure of the grid. The items in the grid conform to the structure. Try each of the options in this exercise. You'll see that

  • The "100px 200px 300px" value creates 3 columns where the left-most column is 100px wide, the next column is 200px wide and the last column is 300px wide. Because these columns all have a fixed width, nothing in the grid changes when you change the width of the browsing window.
  • The "1fr 1fr 1fr" value creates 3 columns of equal width. The fr unit stands for fraction. The more fractional units you devote to a column, the wider it becomes at the expense of the other fractional width columns. Because these columns all have fractional widths, they all change width when the user changes the width of the grid by changing the width of the browsing window since the grid takes up the entire width of the browsing window.
  • The "2fr 1fr 3fr 1fr" value creates 4 columns where the second and fourth columns have equal widths, the first column is twice as wide as the second column, and the third column is three times as wide as the second column. Again, because these are all fractional widths, their widths change as the width of the browsing window changes.
  • The "200px 2fr 1fr" value creates 3 columns where the first column is 200px wide and the second and third columns divide the remaining width of the grid so that the second column is twice as wide as the third column. Since only the second and third columns have fractional widths, only the widths of the second and third columns change as the width of the browsing window changes. The first column remains 200px regardless of the width of the browsing window.
  • The "repeat(6, 1fr)" value creates 6 columns of equal width. "repeat(6, 1fr)" is a short-hand for "1fr 1fr 1fr 1fr 1fr 1fr".

Also, try the gap property to easily separate the rows and the columns by a fixed dimension.

▸The second method uses a naming scheme to specify the relative size and position of two-dimensional regions within the grid. This enables specific columns in a row to span more than one column width and specific rows in a column to span more than one row. It builds upon the simple grid structure in the first exercise and "merges" specific rows and columns.

HTML:

<div id="flex">
<div id="navBar">Navagation</div> <div id="sideBar">Side Bar</div> <div id="item1">Item 1</div> <div id="item2">Item 2</div> <div id="item3">Item 3</div> <div id="content">
<p>Content goes in here.</p> <p>Content goes in here.</p> <p>Content goes in here.</p> <p>Content goes in here.</p> <p>Content goes in here.</p>
</div> <div id="footer">Footer</div>
</div>

CSS:

#navBar {
background-color: purple; grid-area: nb;
} #sideBar {
background-color: darkorange; grid-area: sb;
} #item1 {
background-color: navy; grid-area: i1;
} #item2 {
background-color: darkslategray; grid-area: i2;
} #item3 {
background-color: darkred; grid-area: i3;
} #content {
background-color: green; grid-area: co;
} #footer {
background-color: red; grid-area: fo;
} #grid {
background-color: gray; padding: 10px; display: grid; grid-template-columns: repeat(12, 1fr);
grid-template-areas:
gap: ;
}

Here again, we give a value of grid to the CSS display property of the parent element, and we have designated 12 columns of equal width by setting the grid-template-columns property to repeat(12, 1fr). We have also assigned area names to the elements in the CSS using the grid-area property. This allows to easily define 2-dimensional grid-template-areas using a clever intuitive syntax that suggests the spacial arrangement we seek on our grid. Each row in the layout is represented with a string (in quotes) of element names separated by spaces. Each column of each row is designated with a named area. Each named item is given a rectangular area that is reflected in the grid layout.

This scheme allows us to designate each area across multiple rows and columns. In both radio button options the grid is divided into 4 rows and 12 columns. In the first option the Side Bar spans the first two columns and the middle two rows. In the second radio button option Item 1 spans the first two columns and the last three rows while the Side Bar spans the last two columns and the first three rows.
